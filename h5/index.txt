                    <!-- large代表这个图片可以点击放大 -->

<p>Hickory Pit烧烤店位于加州圣何塞市的一个不起眼的街区，看上去不像一个适合推心置腹的好地方。餐厅外立面是混凝土外壳，里面别有洞天，颇有上世纪50年代的风格：一排排的小摊点装饰着糖果粉和蓝色的塑料，贴面台面，荧光灯。</p>

<p>斯蒂夫•沃兹尼亚克(Steve Wozniak)看上去很自在。这并不奇怪，因为他最近在这里逗留了很长时间。这儿不大好找人，一不小心就会错过这位硅谷传奇工程师。一位服务员指着大厅的后面，但是我只看见两个男人弯着腰，摆弄着两个连在一起的手持设备。他们刚刚用完的聚苯乙烯餐盒放在桌子上，里面还有吃剩的肋条。</p>

<p>服务员坚持说沃兹(大家都这么叫他)就在那里，我再次过去看了一眼，原来其中一个玩手持设备的人就是他。只是他已经吃过了。沃兹尼亚克连忙解释道，他和他的朋友正在玩俄罗斯方块。他问我知道他曾经是俄罗斯方块的世界冠军吗？又问我玩过打砖块游戏breakout吗？这是他为雅达利(Atari)公司设计的，那时候电子游戏还不是用软件写的，是直接烧制到机器的硬件里。他马上又开始描述早期电路的工作原理，回忆他当年花了四天四夜终于把所有的点位都放到屏幕上正确的位置。</p>

<p>“你得把所有芯片上的线路都连好，这样芯片就会交替地出现一、零、一、零、哒、哒、哒、哒”他一边敲，一边说，仿佛回到了上世纪70年代。</p>

<p>这个沉醉其中的工程师须发已经略有斑白，胸毛从黄色的T恤上露出来。他就是苹果神话的另一半。虽然乔布斯(Steve Jobs)更有名，但是沃兹尼亚克一样也是硅谷的传奇。</p>

<p>他曾经是——现在也还是——技术宅男中的极品，硅谷程序员心目中永远的大胡子巫师。计算机工业早期的辉煌，正是源自他少年时对电子的热爱。1976年，他单枪匹马设计的苹果二代(Apple II)，被很多人认为是近几十年来最出色的工程成就，开创了台式机和笔记本电脑进入千家万户的新时代。他和乔布斯也因此成为大明星和千万富豪，几乎在一夜之间开启了个人电脑的革命。</p>

<p>我们坐在一起是因为沃兹尼亚克最近写了回忆录。准确地说，是他口述，一位新闻界的朋友吉娜•史密斯(Gina Smith)执笔。他说自己已经连续50天坐在同样的位置和史密斯一同探讨文字了。听到这儿，我脑海里马上浮现出了两人吃过的、堆得像小山一样的肋条。</p>

<p>那乔布斯对沃兹尼亚克描述的这段硅谷历史怎么看呢？乔布斯比沃兹尼亚克小四岁，他们第一次见面的时候，乔布斯还在上高中。“根据我的理解，他读了之后，觉得我把他写成了一个混球。”沃兹尼亚克说到。</p>

<p>我理解乔布斯的心情。沃兹尼亚克在《我是沃兹：从宅男程序员到偶像》中提到一件事：在经过多个日夜的奋战之后，沃兹终于设计出了史上第一款大热的电子游戏Breakout，乔布斯把这个游戏以700美元卖给了雅达利公司，分给了沃兹一半。后来沃兹发现雅达利实际上给了乔布斯几千美元，乔布斯欺骗了自己的朋友。</p>

<p>沃兹尼亚克也不是特别在乎乔布斯的感受。“史蒂夫有时候挺招人烦的，”他回忆起两人在苹果合作了十年，直到1985年分别离开了苹果。“他有时很讨厌。别人开会的时候他会突然走进来说，得了吧，都是一堆垃圾，你们别做这个了。然后他就走出去，扭脸又说，你们都是傻逼。”</p>

<p>为了防止读者误解沃兹尼亚克的意思，我得申明一下他说话一贯带有工程师固有的直率，甚至天真。他曾经抱怨被断章取义，不过这是很自然的，因为他时常口无遮拦。</p>

<p>为了找补回来，他又谈起了书中的内容，“我确信我在书中说了一些乔布斯的好话，不过我记不住了。”实际上他记得，不过这不足以抵消他对乔布斯的嘲弄。
服务员已经给我们的凉茶添了三次水了，我们再也不能拖着不点菜了。来这种地方，你就顾不得自己的胆固醇水平了。我本来想要一个整份肋排，还是作罢了，只点了一个小份。沃兹尼亚克想点一份甜橙派，但是服务员说没有了，他失望之情溢于言表，只好要了樱桃派。</p>

<p>当你听沃兹尼亚克说话时，你很快就会意识到他跟上了年纪的摇滚明星没什么两样，都渴望重温过去的辉煌。在我们见面的头一个小时中，他不厌其烦地回顾了从11岁开始时做过的各种电子项目：电阻器、闪烁的灯泡和焊接线。</p>

<p>他是工程师中的工程师。在他心目中，历史上的所有大事小情都是他这样的人做出的技术选择导致的直接和唯一结果。如果你问任何一个商科学生，苹果为什么没有保持住早期在个人电脑的领先优势，你会被告知是因为没有把技术授权给其他电脑制造商，这个商业决策的失误给微软留下了机会，他们为“开放”的IBM个人电脑开发操作系统软件，赢得了市场。但是在沃兹尼亚克看来，苹果失败的唯一原因是苹果二代（他的杰作）的下一代产品在技术上的缺陷。</p>

<p>在他的描述中，他开发的电脑是电子行业黄金时代的巅峰之作。那个时候，一个人秉持着做出最好作品的理念，就可以改变整个计算机行业的历史。他做出苹果二代只花了三个月时间。如果一个人就能做出这样的成就，那么接下来的这么多年科技行业上为什么再无可以媲美的建树呢？</p>

<p>“我们在制造电脑的时候，总是认为人比技术更重要，”他说，这样就产生了很多bug和设计糟糕的产品，制造出来的机器对于用户错误操作的各种可能性也考虑不周。“我很确定，苹果和世界上其他公司一样，都失去了成功的秘诀。”</p>

<p>哪儿出了问题？在沃兹尼亚克有些过于简单的世界观看来，是由于工程师的主导地位被商人取代了。个人电脑行业开始进入低迷的原因？“没有好的、扎实的测试体系。没有好好关注产品。基本上是商业利益说了算。”</p>

<p>菜上来了，肋条的味道出人意料地好。配菜是橡木烤玉米和烟熏豆子，却令人难以下咽。我羡慕地看着沃兹尼亚克盘中厚厚的樱桃派。</p>

<p>沃兹尼亚克逐渐进入主题。在他的故事中，工程师是唯一的英雄，是缔造者。这个人“想出了所有的主意，写出了程序，把它展示给别人，又添加画龙点睛的奇妙功能”。</p>

<p>在这个理想的世界中——这仍是硅谷很多工程师的理想——这位英雄工程师还是一个艺术家。</p>

<p>“这就像是我身体的一部分，” 沃兹尼亚克说，“它是一种艺术。当它成为我的一部分，我就会尽力让它变得完美。我记得当我[为苹果二代]做软盘板的时候，我自己来把它固定到主板上，总是搞不定。我连续干了两周，每天都干到凌晨四点。当我终于做完的时候，我意识到如果我用另外一种方法来设计电路，主板上只需要有五个洞，而不是八个。所以我把所有的东西都拆了，从头做起。用户是看不见洞的。但是这对我来说很重要。这是我的一部分。就好像我的身体一样。”</p>

<p>“在那个时候，事情就是这样做的。”</p>

<p>这似乎有点自我神化的成分。但是如果这是一种幻觉，硅谷的人似乎都信这个：一个工程师加一个好主意就可以改变世界。沃兹尼亚克自己就是活生生的例子。金钱可以改变庞大产业如个人电脑的格局，但是不能阻止下一个英雄工程师实现他的梦想。</p>

<p>“尝试和寻找新东西是人的天性，人的头脑就是干这个的，”沃兹尼亚克说，“我们自己天生就有好奇心。可以让世界变得更好的新想法无处不在，但是你拿一大笔钱给一个聪明的工程师，他倒不一定能做出好的东西。”</p>

<p>现在是以谷歌为首的这一代公司大显身手的时代了。对于沃兹尼亚克个人来说，当初的那种臻于完美的技术状态再也没能重现。乔布斯后来回到了苹果，引领了苹果公司的强劲复苏。而沃兹尼亚克也尝试过一些别的事业，比如万能家用遥控器、宠物和个人物品追踪器，都以失败告终。</p>

<p>这些年来，他从事过音乐会推广和慈善工作，有八年时间给10岁的孩子们上课。对于这段时光，他声称自己很满意。不过他还是非常渴望回到个人电脑革命的中心舞台，毕竟他是最早的发起者之一。</p>

<p>“我很希望能参与一些[苹果公司的]事情。但是我觉得史蒂夫不会喜欢这个，”他也承认自己对计算机系统设计的知识已经有些跟不上时代了。这让人很难不产生这样的感觉：尽管这两人的过去是那样不可分割，沃兹尼亚克现在的角色更像偶然造访的同窗旧友或寒酸亲戚，略让人尴尬；你得强装笑容来敷衍他。</p>

<p>关于他最近创办的一个企业，他说，“一个朋友跟我说，嘿，你得把史蒂夫放到董事会，或者让他帮点忙。”乔布斯很快就回应了：“不。”乔布斯也受邀为沃兹尼亚克的新书撰写前言，还是拒绝。</p>

<p>下午4点，午餐时间早过了。自承为“工具控”的沃兹尼亚克起身到餐厅的后门取他的赛格威(Segway)电动车。这是一种自我平衡的两轮电动车，一度被硅谷人士捧为将要改变交通格局的机器。多年过去了，这仍然只是一个新奇的玩具。</p>

<p>现在还没到日落的时候，但是看着沃兹尼亚克站在这个奇怪的机器上准备西行，夕阳西下的感觉油然而生。</p>

                    “一带一路”看着诱人 分羹却不易

                <div class=lead>
                    FT记者休斯：在“一带一路”计划推动下，中国加速布局跨境基础设施建设，预计会带来巨大投资机会。但银行和长期投资者们究竟能在其中扮演什么角色，却尚未可知。

                    2015年06月26日
                </div>
                <div class="inlinevideo" id="DCE25ED3B39457A09C33DC5901307461" autostart=false title="【音频】人民币国际化：借道“一带一路”？" image="http://i.ftimg.net/picture/5/000053645_piclink.jpg" vid="1675" vsource=""></div>

            <div class=byline>珍妮弗•休斯</div>
            <p>中投公司(China Investment Corporation)首席策略官周元最近在香港的一次会议上对与会者说，尽管他不能保证会向大家透露什么情报，但他希望大家会对他说的东西感兴趣。这位冷幽默风格的首席策略官后来谈到了中国对该地区基础设施投资的看法，可以说是既让与会者了解到了一些情报，又引起了他们的浓厚兴趣。</p>
            <p>在亚洲金融圈里，鲜有话题比基础设施更能引起人们的兴趣和争论。与会者想知道，新成立的、中国主导的亚洲基础设施投资银行（Asian Infrastructure Investment Bank，简称：亚投行）到底将如何运作，以及银行、长期投资者等其他群体可能会扮演什么角色。</p>
            <p>最重要的是，很多人都在估算未来几年亚洲需要在基础设施上投入多少，并猜测中国会为此提供多少资金、以及基于什么条件提供。</p>
            <p>“如果你不能指望获得回报，也就是说，如果你不能指望基础设施项目在将来某个时刻向你支付红利，那你就不能再把它称作投资。你还不如把它称作马歇尔计划(Marshall Plan)，”周元在英国《金融时报》投资管理峰会(FT Investment Management Summit)上表示。</p>
            <p>当然有很多人拿中国的亚投行与1948年由美国供资的马歇尔计划作比较。马歇尔计划支持了欧洲在二战后的重建，并且有效地利用了美国的经常账户盈余来支撑美国的经济和地缘政治目标。</p>
            <p>中国目前的计划包括“一带一路”、在金砖国家开发银行和亚投行中的出资，以及其他投资——比如由其主权财富基金中投公司供资的投资。</p>
            <p>中国国家开发银行(China Development Bank)估计，“一带一路”的项目储备库，涉及投资资金超过8900亿美元。其他数字则显示，亚洲直至2020年的基建总投资需求接近8万亿美元。汇丰(HSBC)亚洲项目和出口融资主管詹姆斯•卡梅隆(James Cameron)表示：“每个人给出的数字都不同。只有一件事是肯定的，这些数字全是错的、但它们全都很大。”</p>
            <p>回到具体问题上，银行及其他投资者想知道中国计划中的支出会给他们带来什么机会。项目金融家的日子已然不太好过，因为监管机构正迫使银行为在风险较高的新兴市场的长期放贷留出更多资本金。</p>
            <p>投资者的日子也没有好过多少。对一名物色资产以匹配长期负债的养老基金经理来说，投资越南的一座发电站并以该电站未来的收益作为担保，似乎是个不错的押注。但监管机构担心汇率和国家风险，因此他们要求留够资本金并设立其他监管保障，从而推高了这类投资的成本。法国巴黎银行(BNP Paribas)亚太金融机构团队主管弗朗索瓦•勒布朗(François Leblanc)说：“养老基金和保险公司都非常感兴趣，但或许不总是能够投资。”他认为基金自身的投资准则也构成了限制。</p>
            <p>他说：“困难很多，要填补如今市场供给与投资者需求之间的缺口并不容易。”</p>
            <p>考虑到这一点，周元在商言商（至少是从中投立场出发）的评论应该会受到欢迎。近来有一种担忧是，中国会坚持供应商融资模式，即许多拥有自然资源的发展中国家得到资金来搞基建，而中国企业则是这些基础设施的供应商、资金提供方和建设方。</p>
            <p>如果中国以更认真的态度建立一种在商言商的市场化机制（包括通过其主导的亚投行），那么亚洲就可更普遍地从中获益。</p>
            <p>“亚投行拥有一系列潜在的工具，”汇丰的卡梅隆说，“他们可以直接投资或拨款，考虑或许承担部分风险，提供担保，或帮助确保提前给出正确的建议，以便将项目安排得更具吸引力，好吸引外部融资和投资。有很多工具可帮助开发整个基础设施融资领域。”</p>
            <p>亚洲基础设施格外不好做的原因是，该地区的法律体系和市场多种多样，而且都处于不同的发展阶段。进行当中的项目的范围之广，让这块业务变得更加复杂。但任何走向标准化的迹象——比如一个发电站或道路系统使用此前某个成功项目使用过的融资模式——都将有助于下一个项目。这最终可能会改变亚洲的融资及基础设施的面貌。</p>
            <p>译者/何黎</p>

                            lineWidth: 1

                    negativeColor: '#910000',
                    borderColor: 'silver'
                }
            }
        };
        options = Highcharts.merge(defaultOptions, options);

        return new Highcharts.Chart(options, callback);
    };

    var start = +new Date(),
        $tds = $("$gamecontent td[data-sparkline]"),
        fullLen = $tds.length,
        n = 0;

    // Creating 153 sparkline charts is quite fast in modern browsers, but IE8 and mobile
    // can take some seconds, so we split the input into chunks and apply them in timeouts
    // in order avoid locking up the browser process and allow interaction.
    function doChunk() {
        var time = +new Date(),
            i,
            len = $tds.length,
            $td,
            stringdata,
            arr,
            data,
            chart;

        for (i = 0; i < len; i += 1) {
            $td = $($tds[i]);
            stringdata = $td.data('sparkline');
            arr = stringdata.split('; ');
            data = $.map(arr[0].split(', '), parseFloat);
            chart = {};

            if (arr[1]) {
                chart.type = arr[1];
            }
            $td.highcharts('SparkLine', {
                series: [{
                    data: data,
                    pointStart: 1
                }],
                tooltip: {
                    headerFormat: '<span style="font-size: 10px">' + $td.parent().find('th').html() + ', Q{point.x}:</span><br/>',
                    pointFormat: '<b>{point.y}.000</b> USD'
                },
                chart: chart
            });

            n += 1;

            // If the process takes too much time, run a timeout to allow interaction with the browser
            if (new Date() - time > 500) {
                $tds.splice(0, i + 1);
                setTimeout(doChunk, 0);
                break;
            }

            // Print a feedback on the performance
            if (n === fullLen) {
                $('#result').html('Generated ' + fullLen + ' sparklines in ' + (new Date() - start) + ' ms');
            }
        }
    }
    doChunk();
}
